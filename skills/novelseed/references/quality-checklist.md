# 质量检查与审查体系

来源：snowflake-fiction quality-check + boring-detect + opening-check

## 五维质量评分

| 维度 | 权重 | 检查内容 |
|------|------|----------|
| **冲突强度** | 25分 | 有明确对立力量？冲突在升级？不是伪冲突？ |
| **情绪密度** | 20分 | 能让读者哭/笑/愤怒/紧张？情绪有变化？ |
| **期待感** | 20分 | 读者想知道接下来发生什么？有悬念在牵引？ |
| **节奏控制** | 20分 | 有张有弛？高潮前有铺垫？不拖沓也不仓促？ |
| **钩子设计** | 15分 | 结尾让人必须翻下一页？ |

### 各维度详解

**冲突强度（25分）**
- 5分：毫无冲突，纯流水账
- 10分：有冲突但软弱（内心挣扎为主）
- 15分：有外部冲突但不够尖锐
- 20分：冲突尖锐，双方势均力敌
- 25分：冲突层次丰富，升级路径清晰

**情绪密度（20分）**
- 不是字数多情绪就多。关键看「情绪变化次数」。
- 一段500字的对话，情绪从紧张→释然→再紧张，密度高于2000字的平铺。

**期待感（20分）**
- 核心问题：读者翻页是因为「想知道」还是「习惯性」？
- 每一章应该抛出1-2个新问题，同时回答1个旧问题。

**节奏控制（20分）**
- 快（战斗/对话/冲突）与慢（描写/内心/铺垫）交替
- 连续3章以上同节奏=读者疲劳

**钩子设计（15分）**
- 详见 `references/hook-techniques.md`，13种钩子类型

## 流水账专项检测

### 检测信号
- 连续3段以上纯事件罗列
- 无冲突、无情绪、无转折
- 时间顺序单调推进（"然后...然后...然后..."）
- 信息密度低（一段话只有1个有效信息）

### 修正公式

```
流水账：
  他起床 → 洗漱 → 吃早饭 → 出门 → 上班

修正后：
  闹钟响到第三遍他才从床上爬起来。
  然后他看见了镜子里的自己——双眼布满血丝，但不是因为没睡好。
  昨晚那通电话的内容，他一个字也想不起来了。
  早饭？他完全忘了吃。
```

公式：事件 → **意外/冲突** → **情绪反应** → **悬念钩子**

## 开篇质量检查（黄金三章专用）

| 检查项 | 通过标准 | 不通过处理 |
|--------|----------|------------|
| 前300字爆点 | 有冲突/意外/悬念，不是背景介绍 | 直接把冲突前置 |
| 主角登场 | 前500字出现，通过事件展示核心性格 | 删掉非主角开篇段落 |
| 紧迫目标 | 主角有明确想做/必须做的事 | 给主角一个72小时内必须完成的目标 |
| 阻碍出现 | 第1章内有障碍物/敌对力量 | 立即加入一个具体阻碍 |
| 小爽点 | 第1章内≥1个小爽点 | 加入打脸/发现/小胜利 |
| 结尾钩子 | 必须！不可省略！ | 见hook-techniques.md选一个 |
| 信息密度 | 每500字≥3个有效信息 | 删废话，加信息 |
| 节奏 | 不拖（前500字进入主题） | 砍掉环境描写、背景介绍 |

### 常见开篇错误

1. **开场堆设定** — "这个世界有三大帝国、五大种族..." → 删
2. **主角在睡觉/做梦** — 最无聊的开头，等于没开头
3. **天气描写开篇** — "今天是晴天" = 读者已走
4. **对话开头但无信息** — 聊天不是冲突
5. **回忆开头** — "他想起三年前..." = 读者没耐心

## 全稿连贯性检查

| 检查项 | 方法 |
|--------|------|
| 人设一致性 | 主角核心标签在全文行为中是否始终如一 |
| 时间线 | 各事件日期间隔是否合理 |
| 伏笔回收 | 列出的伏笔是否都有后续呼应 |
| 力量体系 | 战力/等级是否有前后矛盾 |
| 地名/人名 | 是否有拼写不一致 |

## 使用方式

对任意章节说：
```
/检查质量 第X章
/检查流水账 第X章
/检查开篇
/检查连贯性
```
