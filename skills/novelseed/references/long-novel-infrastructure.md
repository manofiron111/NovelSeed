# 长篇连载小说基础设施

> 76章以上长篇的核心风险：「写到第40章时不记得第12章的细节」。
> 解决方案：把记忆外包给系统。经过长篇小说实战验证。

## 一、72项设定清单

一部长篇小说需要的全部设定。缺了的迟早会变成剧情矛盾。

### 元数据（1-6）
书名、类型/题材、目标平台、读者定位、总字数目标、一句话卖点

### 世界观（7-15）
时代、地点、社会背景、力量体系、世界规则、历史大事件、组织/势力、关键物品、核心追问

### 人物设定（16-33）——每人一套
姓名、年龄、性别、外貌、性格、身份、出身、专长、**弱点**、口头禅、欲望、**恐惧**、成长弧、人际、秘密、**底线**、**说话方式**、参考原型

> 🔑 弱点/恐惧/底线/说话方式——最容易漏，也最容易导致人设崩塌。

### 剧情设定（34-44）
核心冲突、主线、驱动问题、支线、叙事结构、各卷冲突、转折点、高潮、结局、节奏、反转

### 风格设定（45-53）
语言、视角、基调、对话、描写、术法/战斗、情感、幽默、悲剧底色

### 结构设定（54-60）
分卷、分章、每章字数、标题格式、章结构、黄金三章、卷结构

### 知识体系（61-66）
传播顺序、每卷知识点、典籍、深度控制、传播方式、准确性校验

### 追踪系统（67-72）
时间线、伏笔、出场记录、知识已传、发布状态、人物快照

---

## 二、双层架构：技能 + 档案馆

```
技能 (10-15KB) = 导航层 → 速查摘要 + 指针
档案馆 (80KB+)  = 数据层 → 详细设定 + 追踪 + 正文
```

**技能里存什么**：行为规则、角色一句话、卷章一行、目录树
**档案馆存什么**：所有详细设定、追踪文件、章节正文

档案馆模板（已验证可用）：
```
archive/novels/<书名>/
├── README.md                   # 索引
├── 00-synopsis.md              # 简介
├── 01-world-setting.md         # 世界观
├── 02-characters.md            # 全角色档案
├── 03-plot-mainline.md         # 主线
├── 04-volume-structure.md      # 卷章结构+回目+钩子+节奏
├── 05-artifacts.md             # 物品/法器
├── 06-writing-rules.md         # 写作规范+自检
├── 07-metaphysics-map.md       # 知识传播映射
├── 08-romance-arc.md           # 感情线
├── 09-villain-arc.md           # 反派弧线
├── 10-dramatic-irony.md        # 反讽设计
├── 11-exit-plan.md             # 退场计划
├── 12-platform-metadata.md     # 平台元数据
├── 13-reader-knowledge.md      # 读者知识追踪
├── 14-sop.md                   # 每章SOP
├── 15-retrospective-template.md # 卷回顾
├── timeline.md                 # 时间线
├── plot-threads.md             # 伏笔追踪
├── character-appearances.md    # 出场矩阵
├── publication-log.md          # 发布记录
└── chapters/volN/chN.txt       # 正文
```

## 三、每章SOP

**写前（5分钟）**：查回目→时间线→伏笔→角色位置→弱点/恐惧/底线→读者知识→玄学概念

**写中**：目标风格/字数1500-2500/新概念≤2

**写后（8分钟）**：13项自检→存档正文→更新8个追踪文件→技能进度

**定期维护**：每10章伏笔清理 / 每卷末回顾报告 / 第38章中点审计

---

## 四、钩子轮换 + 节奏轮换

13种钩子，每两章不重样：疑问/行动/反转/情感/时间/人物/对话/知识/威胁/选择/代价/视角/沉默

三档节奏，连续不超过2章同档：快(~50%)/中(~20%)/慢(~30%)

---

## 五、关键教训

1. 技能不能存详细数据——34KB→13KB精简是必要的
2. 完美角色=扁平角色——必须有弱点/恐惧/底线
3. 跳过一次写前检查→后面必出矛盾
4. 交叉引用必须审计——标题说A内容说B会被读者发现
